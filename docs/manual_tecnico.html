Documentación de Backend y Escalabilidad de Servicios

1. Arquitectura de Repositorios y Servicios

El sistema implementa el patrón Repository para desacoplar la lógica de persistencia de los controladores. Esto permite la interoperabilidad entre SQLite y PostgreSQL sin rediseñar el core del sistema.

  • BaseRepository: Clase abstracta para operaciones CRUD básicas.
  • PharmacyService: Orquestador de lógica de negocio (Vencimientos, Alertas).

2. Pila Tecnológica (Stack)

Capa Tecnología Propósito
Servidor PHP 8.2 (Render/WAMP) Motor de Scripts
Persistencia PostgreSQL / SQLite Almacenamiento de Datos
Frontend JavaScript (WAAPI) / CSS3 Experiencia UX Élite

3. Sincronización Cloud

El despliegue se realiza vía GitHub Actions / Render Webhooks. La base de datos en producción reside en Supabase, garantizando alta disponibilidad para toda la red de la ESE Fabio Jaramillo Londoño.
